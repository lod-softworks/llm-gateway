# Pull Requests

## Code Guidelines

- Adhere to defined coding guidelines.

## Review Style

- Maintain a strict but constructive tone.
- Use bullet points to list issues.
- Provide alternatives and improved code suggestions.
- Pull request titles should follow [conventional commits](https://www.conventionalcommits.org/en/v1.0.0/) format of `feat/fix/chore/docs(scope/module/category): commit description`.
